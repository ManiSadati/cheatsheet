# Repository Rules

This repository stores engineering learning notes and experiments.

## Notes
- Use Markdown.
- Keep notes concise.
- Prefer sections: Problem / Idea / Example / Related.

## Experiments
- Every experiment must have a README.md.
- Include: Goal, How to Run, Results.

## Code
- Prefer small self-contained examples.
- Languages: C, C++, Python.

## Organization
Topics are grouped by directory:
- compilers
- mlir
- performance
- llm-systems
